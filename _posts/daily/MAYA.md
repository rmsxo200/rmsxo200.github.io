**- 기타 -**

  

섭디비젼 : 지금은 망한기능, 알필요 없으면 3번 버튼으로 남아있다.

ALT + 마우스 R : 확대, 축소

     Ease in (slow in) : 가속

  

     Ease out (slow out) : 감속

     Moving-hold :

          멈춰있는듯 하지만 미세하게 움직이는 동작

          사람이 가만히 있어도 살아있는듯한 느낌을 주기 위해 움직임을 준다. (숨쉬는 동작, 눈 깜빡임 등)

          역방향 무빙홀드 : 이전그래프와 진행방향이 반대일때 (쿠션과 비슷한 효과, 회복하는 느낌)

정방향 무빙홀드 : 이전그래프와 진행방향이 같을때

  

     hold : 0값, 그래프의 움직임이 없을때

  

     Anticipation :

          무언가 움직이기전 준비동작으로 그래프가 똑바로 상승하지 않고 중간에 역방향으로 갔다가 상승한다.

          예) 점프전 몸을 움츠리는 동작, 야구방망이를 휘두르기전 손이 뒤로 가는 동작등

  

     Cusion : 

          원래대로 돌아오는 작용.

          진행방향으로 갔다가 다시 회복하는 움직임이다.

          예) 점프후 착지할때 움츠려있던 몸이 다시 일어서는 동작, 기지개를 쫙펴는 동작 후 다시 돌아오는 동작, 주먹으로 펀치를 날리고 팔을 뻗은 후 살짝 다시 굽혀지는 동작 등

          움직임이 멈출때는 Ease out 혹은 Cusion을 사용한다.

  

     키포즈만 작업이된 상태에서 중간에 Ease in, Ease out 을 넣는 방법 :

          키포즈를 선택후 마우스 가운데 버튼으로 키포즈에서 3~4프레임 뒤에 전체 키를 준 후 위치를 더 뒤로 움직이면 그래프 곡선이 Ease in이 된다.

          Ease out 은 반대로 키포즈에서 3~4프레임 앞에 전체 키를 준 후 더 앞으로 움직이면 된다.

          스냅과 이즈인, 이즈아웃을 어느정도로 설정해야 되는지 모르겠으면 6:4:6 의 비율로 작업한다. (이즈인 6, 스냅 4, 이즈아웃 6)

아래의 타임라인에서 마우스 휠을 누른채 움직이면 화면은 고정된 채로 프레임만 움직인다.

  

*카메라에 키를 줘 화면이 흔들리는 효과 연출 가능

  

*Components 모드에서 점, 선, 면에 키를 줄때는 S키를 눌러 키를 줄수있다.

  

Visualize > Ghost Selected : 키를 준오브젝트가 움직인 자리에 잔상을 남겨 그림자들의 너비를 비교하여 키를 잘줬는지 확인할 수 있다.

  

캐릭터를 움직일때 구부리는 동작이 있으면 전체적인 모습이 아크(ARC, 호모양곡선) 모양으로 되야한다.

  

IK : 하나의 관절을 움직이면 가운데 있는 다른관절도 같이 움직인다.

걷기 (Walk Cycle) :

     보폭이 클수록 발 뒷꿈치가 많이 떨어진다.

     땅에 닿아있는 발에 무게중심이 생기고 그쪽 방행으로 몸이 기울어진다.

     걸음이 느리면 몸이 더 많이 기울어지고 빠르면 별로 기울어 지지 않는다.

     기본 워킹은 6프레임단위로 한다. (0에 패싱포즈, 6에 컨택포즈, 12에 패싱포즈, 18에 컨택포즈, 24에 패싱포즈)

패싱포즈 : 걸을때 머리가 가장 높게 올라가는 포즈. (Up 포즈, 지나가는 포즈)

  

컨택포즈 : 걸을때 머리가 가장 낮게 되는 포즈. (Down 포즈)

     3프레임 - 경보, 달리기

     10프레임 - 피곤하게 걷기, 늙은 사람 걷기

     12프레임 - 조심스럽게 걷기

**- Parent -**

  

Edit > Parent :

     여러개를 선택한 후 누르면 마지막에 선택한게 부모로 설정

     Windows > outliner 에서 해당 오브젝트를 마우스 휠로 드레그해 두개를 겹쳐도 부모자식관계 형성

Constrain > Parent :

     먼저 선택한게 부모가 되고 두번째 선택한게 자식이 된다.

     자식은 부모를 따라서 움직이고 부모를 여러개 지정할 수 있다.

     자식 오브젝트는 우측 Channel Box에 설정되있는 부모오브젝트들이 보이며 '0','1' 값을줘  영향을 받거나 받지 않게 할 수 있다.

     또 해당 항목에 키를 줘 일부 프레임에서만 부모에게 영향을 받도록 설정할수도 있다.

![](HTML%20import/Attachments/제목%20없음%20(104).png)

     위그림의 항목에 설정한 키는 타임라인에는 보이지 않고 그래프에서 확인할 수 있다.

     키 설정시 한 프레임 안에 수정해야 딱딱 맞게 움직인다.

     부모자식은 처음 설정할때 거리가 계속 유지된다.

     자식으로 설정된 오브젝트는 독립적으로 키를 줘 움직일수 없다.

![](HTML%20import/Attachments/제목%20없음%20(48).png)

**-리깅-**

  

*조인트와 연결한 Locator 나 Curve는 Freeze한 후 사용해야 한다.

조인트의 Trenslate 값은 프리즈가 안되기 때문에 손잡이에 Freeze를 하는 것이다.

  

Skeleton > Create Joints : 새로운 뼈를 생성한다. 생성시 Shift를 누르고 만들면 직선으로만 움직여 일자로 생성할 수 있다.

  

Skeleton > Insert Joints : 두개의 뼈 사이에 새로운 뼈를 추가할때 사용한다.

  

Skeleton > Mirror Joints : 선택한 뼈를 XY,YZ,XZ축 기준으로 반대편으로 복사한다.

  

Skeleton > Remove Joints : 중간에 있는 뼈를 삭제한다.

  

Skeleton > Disconnect Joints : 선택한 하위에 뼈 전체를 분리해낸다.

  

Display > Animation > Joint Size : 조인트의 크기를 조절한다.

  

D > 중간에 있는 뼈만 움직일 경우 D키를 누르고 움직이면 된다.

  

Skin > Bind Skin :

     오브젝트와 뼈를 선택한뒤 명령어를 실행하면 뼈와 연결되어 오브젝트가 같이 움직인다.

     바인드 후에는 오브젝트와 뼈 둘다 수정할수 없다.

     수정시에는 해제 후 진행해야 한다.

  

Skin > Unbind Skin :

     바인드를 해제한다.

**-다이나믹-**

  

다이나믹 사용전 환경설정에서 Timee Slider > Playback > playback speed 를 Real-time[30 fps] 에서 Play every frame으로 변경한다.

  

리지드는 딱딱한 고정된 물체를 표현할때 사용한다.

  

Channel Box에 다이나믹 속성 :

     Mass = 무게

     Bounciness = 탄력(통통튀는)

     Damping = 감속

     Static Friction = 마찰

     Stand In = cube 나 sphere로 변경하면 복잡한 오브젝트도 단순 원이나 사각형으로 인식해 프로그램이 가벼워진다. 하지만 주변 면적이 늘어나 오브젝트는 겹치지 않았어도 겹친걸로 인식할수 있다.

     Initial Velocity X,Y,Z = 시작 속도 부여

     Initial  Spin X,Y,Z = 시작 회전 값 부여

  

주의사항 :

     면이 뒤집혀 있으면 안되고 노말끼리 부딪혀야 한다.

     시작전 오브젝트끼리 미리 겹쳐저 있으면 않된다.

Fields/Silvers > Create Active Rigid Body :

     움직이는 객체를 설정, 키설정 불가능

     Active 상태의 오브젝트가 가만히 있지 않고 움직이는 경우 채널박스에 Active 속성에 키를줘 처음에 Passive로 있다가 충돌 직전 Active로 바꾸는 방법을 사용한다.

  

Fields/Silvers > Create Passive Rigid Body : 고정되있는 객체를 설정, 키설정 가능

  

Edit > Delete by Type > Rigid Bodies : 리지드를 삭제

  

Fields/Silvers > Gravity : 중력 설정

  

Fields/Solvers > Radial : 중력과 반대로 밀어내는 효과 ,사방으로 밀어낸다.

  

*Gravity 와 Radial 의 중요속성 : Magnitude, Attenuation, Max Distrance

  

Fields/Solvers > Turbulence : 바람이 부는 듯이 흩으리는 효과 (난기류)

  

Fields/Solvers > vertex : 중심부터 회오리 쳐서 돌면서 날려버림

  

Fields/Silvers > Initial State > Set for Selected :

     위치를 세팅한다.

     다이나믹 중간에 오브젝트의 위치를 처음으로 고정시키고 싶으면 해당 명령어를 준다.

     그럼 시작할때 설정한 위치에서 시작하게 된다.

  

리지준걸 움직이기 위해선 다른 오브젝트와 충돌한다던가 중력을 주던가, 채널박스에 초기이니셜 값을 줘야 한다.

  

실행시키기 위해선 재생 버튼을 누르면 된다.

하지만 실제 속도와 다르기 때문에 타임슬라이드를 마우스R 클릭 후 Playblast를 눌러 확인해야 한다.

![](HTML%20import/Attachments/제목%20없음%20(11).png)

  

Fields/Silvers > Create Nail Constraint : 생성 후 쭉 잡아당겨 위치를 이동시켜주면 해당점을 기준으로 오브젝트가 실에 메달린것 처럼 대롱대롱 움직인다.

  

Fields/Silvers > Create Pin Constraint :

     오브젝트 두개를 선택하고 명령어를 실행하면 두개의 오브젝트가 연결된다.

     완전히 붙는게 아니고 체인처럼 연결은 되있지만 독립적으로 움직인다. (사용시 두 개를 초과해 선택하면 안된다.)

     고무줄 같은 느낌이다.

     Nail 처럼 생성 후 쭉 잡아당기며 해당 방향으로 날아간다.

Effects 쪽은 프리셋이다.

  

Effects > Shatter :

     오브젝트를 선택하고 해당효과를 주면 오브젝트가 깨진효과를 낼수 있게 오브젝트를 쪼개준다.

     Surface Shatter = 속이 빈상태로 쪼갠다. Extrude 값을줘 두깨를 설정할수 있다.

     Solid Shatter = 속이 찬 상태로 조각낸다. Edge jagginess 값으로 깨지는 면이 울퉁불퉁하게 할 수 있다.

     Crack Shatter = 판(Plane)에 사용한다.     

     Shatter에서 Rigid body를 줄땐 생성시 옵션에서  Post operation > Rigid bodies with collisions off 항목을 설정한다.

     또 Original surface > Link to shards 로 설정해야 채널박스에서 Shatter라는 항목이 생겨 해당옵션을 키고 끌수 있다. 키를 줄수도 있다.

![](HTML%20import/Attachments/제목%20없음%20(72).png)

  

     Shatter 에서 오브젝트를 쪼갤때 리지드바디를 주면 우측 채널박스에 리지드속성중 Collisions 속성이 'off' 되어 있는데

     이렇게 되면 오브젝트간 충돌하지 않고 통과한다.

     이유는 처음부터 on으로 되어 있으면 쪼개진 오브젝트간 엉켜서 움직이지 않는다.

     그래서 일단 나눠논 오브젝트를 퍼트린후 Collisions를 'on' 하도록 키를 주거나 닿아야 하는 오브젝트에 Collisions Layer값을 '-1'로 설정한다.

     Collisions Layer 값이 '-1'이면 모든 오브젝트에 충돌한다.

  

     **예)** 

  

          **부서지는 벽 : Collisions = off**

                              **Collisions Layer = 0**

  

          **바닥, 투사체 :Collisions = on**

                              **Collisions Layer = -1**

nCloth > Create Passive Collider : Cloth를 준 오브젝트와 충돌하게 해준다. (해당 오브젝트는 고정)

     오브젝트에서 파티클 방출

     Emitter Type을 Omni나 Directional로 맞추면 vertex에서 파티클이 분사되고 Surface로 하면 면에서 분사된다.

  

Emitter 속성 :

     Emitter Type :

          Omni = 파티클을 360도 뿌린다.

          Directional  =

               파티클을 최대 180도 밖에 뿌리지 못한다.

               아래 Direction이 붙은 옵션들을 사용할때 Emitter Type을 Directional로 바꿔놔야 적용된다.

     Rate : 파티클이 나오는양, 파티클의 수

     Direction X,Y,Z : 파티클의 방출 방향

     Spread : 파티클을 퍼트리는 정도 (최대값 1, 180도)

     Speed : 파티클 방출 속도

  

Particle을 선택한 뒤 우측 Attribute Editor에서 particleShape탭을 선택한 뒤 Per particle (Array) Attributes 항목으로 세부 옵션을 설정한다.

  

파티클 색바꾸기 :

**'Condition(조건문)'**

**Maya Render node Utilities** 중에 **'Condition'** 에 대해 알아보겠습니다. 

'Condition' 은 조건 이라는 뜻으로 프로그램 문법으로 '조건문'으로 쓰여집니다.

**조건문(Condition)**  
루틴내에서 주어진 어떤 조건이 만족되었을때만 수행되는 문장입니다.

(루틴(Routine)이란 컴퓨터에서 특정한 작업을 실행하기 위한 일련의 명령으로   
프로그램의 일부 혹은 전부를 이르는 경우에 씁니다.)

조건문 **컨디션**이 가지고 있는 조건은   
1. 같음 ( == )  
2. 다름 ( =! )  
3. 보다 큼 ( >)  
4. 크거나 같음 ( >= )  
5. 보다 작음 ( < )  
6. 작거나 같음 ( <= )

입니다. 그래서

이런한 조건이 충족될때는 'A'라는 일을 수행하라

이런한 조건이 충족되지 않을 시에는 'B'라는 일을 수행하라. 뭐 이런식으로 구성이 되어 있는 문장입니다.

![](HTML%20import/Attachments/Attachment%20(8).png)

아이콘 모양은 이렇게 생겼습니다.

이 문장을 Maya에서는 이렇게 아이콘화 되어 있습니다. (물론, 스크립트, 문자로 써도 상관없습니다.) 

![](HTML%20import/Attachments/Attachment%20(5).png)

컨디션을 클릭하고 어트리뷰트 에디터로 보게되면 위와 같은 내용으로 되어 있습니다.

First Term 은 첫번째 값, Second Term은 두번째값을..

Operation에서 조건을 수행합니다.

![](HTML%20import/Attachments/Attachment%20(1).png) 

조건은 종류는 위와 같은 6개중에 하나입니다.

1. 같음 ( == )  
2. 다름 ( =! )  
3. 보다 큼 ( >)  
4. 크거나 같음 ( >= )  
5. 보다 작음 ( < )  
6. 작거나 같음 ( <= )

 우선 Greater Than 을 선택합니다.

즉 '보다 크다'입니다. 이것은 첫번째 값이 두번째 값 보다 크다 라는 조건을 수행합니다.

그 아래쪽을 보면  Color If True 와 Color If False가 있습니다.

Color If True 는 위의 조건이 참이면 실행하는 것이고

Color If False는 위의 조건이 거짓일때 실행하는 것입니다.

그 옆의 세개의 칸이 있는 데 그곳에 수치를 입력할수 있다.

세개의 칸이므로 이것이 R,G,B도 될수 있고 위치값 x,y,z, 이 될수도 있고 스케일값 x,y,z이 될수도 있습니다.
